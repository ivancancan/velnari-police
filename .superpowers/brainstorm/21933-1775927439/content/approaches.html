3 enfoques para construir Velnari Insights

Elige el que mejor se ajuste al tiempo y alcance que buscas

A

Mejorar /admin/analytics existente

La página ya existe con filtros y datos. Agregarle KPI cards arriba, presets de fecha (Hoy / Semana / Mes / Custom) y reemplazar las barras CSS con Recharts real. Misma URL, misma estructura.

Pros

  • Rápido (~1 día)
  • No duplica código
  • Ya tiene export CSV/PDF

Contras

  • Página ya está llena
  • Sin drill-down ni role-gating
B

✦ Nueva ruta /insights (recomendado)

Sección nueva dedicada. KPI cards en grid arriba — al hacer clic cada card se expande con gráficas Recharts completas. Selector de período con presets arriba. Roles: admin/commander ven todo, supervisor solo el día de hoy.

Pros

  • UX limpia desde cero
  • Drill-down por card
  • Role-gating correcto
  • Fácil de crecer

Contras

  • 2–3 días de trabajo
  • La API ya tiene todos los endpoints
C

Plataforma analytics completa

Todo lo anterior + actualizaciones en tiempo real por WebSocket, comparación período vs período (esta semana vs la pasada), detección de anomalías visible, reportes PDF automáticos por turno y exportación formato SESNSP para gobierno.

Pros

  • Nivel enterprise
  • Diferenciador fuerte
  • Reportes automáticos

Contras

  • 5+ días
  • Demasiado para el piloto

Mi recomendación es B — la API ya tiene todos los datos, solo falta la UI correcta.
